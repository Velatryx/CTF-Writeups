## Best Practices & Sanitization Guidelines

To maintain professional and ethical standards, all contributions and notes within this repository adhere to the following guidelines:

1. **Payload Abstraction:** Active attack commands (e.g., specific command injection hooks or connection strings) should be replaced with benign proof-of-concept placeholders like `echo "Vulnerability Verified"` or `whoami`.
2. **Network Sanitization:** Real-world or target-specific IP addresses must be scrubbed. Use standard documentation or loopback addresses such as `127.0.0.1`, `10.x.x.x`, or placeholders like `ATTACKER_IP`.
3. **Respecting Write-up Embargos:** For active/live CTF competitions, solutions and code will not be pushed to this public repository until the event has officially concluded and the organizers permit public write-ups.
